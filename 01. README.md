# AVD — Adaptive Visual Design

> **A technical standard that transforms optometric parameters into adaptive visual rules for digital interfaces.**

AVD (Adaptive Visual Design) is not a software application. It is a **technical specification and algorithmic procedure** that bridges the gap between clinical optometry and UI/UX design, enabling digital interfaces to adapt dynamically to the real visual capacity of each user.

## 🌍 The Global Need: Why AVD?

Visual impairment is the most widespread health condition on the planet. While traditional glasses correct physical vision, they do not solve the dynamic challenges of digital interfaces. **AVD bridges this gap** by translating clinical optometric parameters into adaptive UI rules.

### 📊 The Scale of the Challenge

According to the [World Health Organization (WHO)](https://www.who.int/news-room/fact-sheets/detail/blindness-and-visual-impairment) and recent epidemiological studies, the need for visual adaptation is not a niche—it is the majority:

| Metric | Global Estimate | Impact |
| :--- | :---: | :--- |
| 👓 **Current Glasses Users** | **~4.0 Billion** | Over 50% of the global population relies on corrective lenses daily |
| 🔍 **Need for Correction** | **1.2B - 2.2B** | People living with uncorrected or under-corrected refractive errors |
| 🚫 **Lack of Access** | **>1.0 Billion** | Suffer from vision impairment simply because they lack access to glasses |
| 📈 **The Myopia Epidemic** | **~5.0 Billion (2050)** | 50% of the world population projected to be myopic by 2050 |

*Sources: [WHO World Report on Vision 2019](https://www.who.int/publications/i/item/9789241516570), [Holden et al. 2016](https://iovs.arvojournals.org/article.aspx?articleid=2166165)*

### 💡 The Digital Gap

Traditional optometric correction (Sphere, Cylinder, Axis, Add) is designed for the static physical world. However, digital screens demand **dynamic adaptation**:

- **Contrast sensitivity management** for varying lighting conditions
- **Spatial frequency optimization** for different screen resolutions
- **Digital eye strain reduction** through adaptive spacing and sizing

Current web accessibility guidelines (WCAG 2.2) provide a solid baseline, but they lack **clinical personalization**. They treat all users with "visual impairment" as a homogeneous group, when in reality, each prescription requires specific UI adjustments.

### 🚀 The AVD Solution

AVD acts as the **missing link between optometry and UI/UX design**. By reading a user's exact prescription (S, C, A, Add), AVD allows interfaces to automatically adjust:

- ✅ Typographic density (font size, letter spacing)
- ✅ Contrast levels (text/background combinations)
- ✅ Spatial distribution (line height, margins)
- ✅ Visual weight (font weight, stroke thickness)

This ensures that digital environments are not just legally compliant, but **clinically optimized** for the actual visual capacity of each user.

## 🧩 What is AVD?

AVD is a **standard**, not an executable program. It defines:

1. **Input format** — The optometric parameters required (Sphere, Cylinder, Axis, Addition)
2. **Normalization rules** — How to homogenize values across different optical systems
3. **Calculation algorithm** — How to derive adaptive coefficients from clinical data
4. **Profile generation** — How to classify users into visual sensitivity categories
5. **Output delivery** — The structured rules (CSS, accessibility parameters, block structure) that any interface can consume

Because it is a specification, AVD can be implemented in:

- Web browsers (via CSS/JS)
- Desktop applications (via Qt/QSS, Electron, etc.)
- Mobile apps (iOS/Android accessibility services)
- Operating systems (system-wide adaptive themes)
- E-readers and assistive devices

## ⚙️ The AVD Algorithm

The algorithm follows **5 sequential steps**:

┌─────────────┐     ┌──────────────┐     ┌─────────────────┐
│  1. INPUTS  │────▶│ 2. NORMALIZE │────▶│ 3. COEFFICIENTS │
│ S, C, A, Add│     │  Parameters  │     │   Calculation   │
└─────────────┘     └──────────────┘     └────────┬────────┘
                                                   │
                    ┌──────────────┐     ┌─────────▼─────────┐
                    │ 5. OUTPUT    │◀────│ 4. VISUAL PROFILE │
                    │ CSS / A11y   │     │   Generation      │
                    └──────────────┘     └───────────────────┘


| Step | Name | Description |
| :---: | :--- | :--- |
| **1** | **Inputs** | Receives the 4 classic optometric parameters: Sphere (S), Cylinder (C), Axis (A), Addition (Add) |
| **2** | **Normalization** | Homologates values for consistency across any optical system (e.g., converts plus-cylinder to minus-cylinder notation) |
| **3** | **Coefficients** | Calculates internal tolerances: contrast sensitivity, size tolerance, spacing tolerance |
| **4** | **Visual Profile** | Generates a summarized profile with sensitivity and density levels (HIGH / MEDIUM / LOW) |
| **5** | **Output Delivery** | Delivers structured rules: CSS directives, accessibility parameters, block structure rules |

## 🚀 How to Use AVD

Since AVD is a specification, integration depends on your platform. The general workflow is:

### 1. Capture the user's prescription
```json
{
  "sphere": -3.50,
  "cylinder": -1.25,
  "axis": 180,
  "addition": 0.00
} ```

2. Run it through the AVD algorithm
Apply the 5 steps defined in the specification to generate adaptive coefficients.
3. Apply the output to your interface
Consume the generated CSS variables, accessibility parameters, or structural rules.

bash

python3 avd_generator.py

Outputs:

    avd_output_example1.css — CSS for moderate myopia
    avd_output_example2.css — CSS for presbyopia
    avd_profile.json — Complete visual profile in JSON format

🔮 Future Work
AVD is an evolving standard. Planned developments include:

    W3C/WCAG alignment — Formal mapping to WCAG 2.2 criteria
    Browser extension — Real-time application of AVD rules on any website
    Optometrist integration — Direct export from clinical software
    Machine learning refinement — Adaptive coefficients trained on real user data
    OS-level integration — System-wide adaptive themes for Windows, macOS, Linux, Android, iOS
